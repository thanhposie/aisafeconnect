MODULE 1: Xác thực & Quản lý Tài khoản (AUTH)
TC-SC-01 Đăng ký tài khoản hợp lệ

  • Nhập username, nickname, password hợp lệ
  • Bấm Đăng ký

TC-SC-02 Đăng ký trùng username
  • Nhập username đã tồn tại
  • Bấm Đăng ký
 Hiển thị thông báo lỗi "Tên đăng nhập đã tồn tại". PPP ✅ API trả 400 + thông báo lỗi đúng.
TC-SC-03 Đăng nhập hợp lệ
  • Mở trang /login
  • Nhập đúng username/password
  • Bấm Đăng nhập
 Chuyển vào trang Home, hiển thị tên người dùng. PPP ✅ Login OK, session được lưu.
TC-SC-04 Đăng nhập sai mật khẩu
  • Nhập đúng username, sai password
  • Bấm Đăng nhập
 Hiển thị "Mật khẩu không chính xác". FFP ❌ Lần 1&2: Form bị xóa trắng nhưng không hiện thông báo lỗi rõ ràng. ERR-01Đã fix lần 3
TC-SC-05 Tài khoản bị ban đăng nhập
  • Admin ban user X
  • User X thử đăng nhập
 Hiển thị "Tài khoản bị khóa bởi hệ thống". PPP ✅ Server trả 403 đúng.
TC-SC-06 Session Gate - truy cập khi chưa login
  • Chưa đăng nhập
  • Truy cập thẳng URL /chat hoặc /profile
 Tự động redirect về trang Login. FFF ❌ Trang /chat load được mà không cần login. Frontend chưa có route guard. ERR-02Cần fix
MODULE 2: Video Chat (WebRTC + Socket.IO)
TC-SC-07 Vào hàng đợi 1-1
  • Login với 2 tài khoản khác nhau (2 tab)
  • Cả 2 bấm "Tìm người chat"
 Server ghép đôi, cả 2 nhận được sự kiện match-found. PPP ✅ Matchmaking hoạt động tốt trong cùng máy.
TC-SC-08 Mutual Approval - cả 2 đồng ý
  • Sau khi match, cả 2 bấm "Đồng ý"
 Kết nối WebRTC được thiết lập, video/audio bật. PEPEPE ⚠️ Cần thiết bị thật để test camera/micro. Chưa test được trên CI. Cần thiết bị thật
TC-SC-09 Mutual Approval - 1 người từ chối
  • Sau match, 1 người bấm "Từ chối"
 Cả 2 nhận match-failed, quay lại trạng thái chờ. PPP ✅ Server emit match-failed đúng cho cả 2 bên.
TC-SC-10 Approval timeout 15 giây
  • Sau match, không ai bấm gì cả
  • Chờ 15 giây
 Server tự huỷ match, emit match-failed với reason "timeout". PPP ✅ setTimeout hoạt động đúng 15s.
TC-SC-11 Ngắt kết nối giữa cuộc gọi
  • Đang video call, tắt mạng hoặc đóng tab
 Người còn lại nhận partner-disconnected, thoát phòng. FPP ❌ Lần 1: UI bị đứng, không hiện thông báo partner offline. Lần 2 trở đi OK. ERR-03Race condition fix

TC-SC-13 Thiếu tham số bắt buộc
  • Gọi POST /api/translate
  • Body: {text: "Hello"} (thiếu targetLang)

  • Gửi liên tiếp 100 request dịch thuật

Mã lỗi TC liên quan Mô tả lỗi Nguyên nhân Cách fix Trạng thái
ERR-01 TC-SC-04 Form bị xóa trắng khi đăng nhập sai nhưng không hiển thị thông báo lỗi. State lỗi chưa được set trong catch block của Login.tsx. Fix: Thêm setError(err.response?.data?.error) trong catch block của hàm handleSubmit ở Login.tsx. ✅ Đã fix
ERR-02 TC-SC-06 Trang /chat và /profile có thể truy cập mà không cần đăng nhập. Frontend chưa implement Route Guard / PrivateRoute. Fix: Tạo component ProtectedRoute trong src/routes/index.tsx. Wrap các route cần auth bằng <ProtectedRoute>, redirect về /login nếu chưa có session. 🔧 Chưa fix
ERR-03 TC-SC-11 Khi partner ngắt kết nối đột ngột, UI VideoChat bị đứng, không hiện thông báo. Event partner-disconnected từ Socket.IO chưa được lắng nghe trong VideoChat.tsx. Fix: Trong VideoChat.tsx thêm:
socket.on('partner-disconnected', () => { setStatus('disconnected'); toast.error('Đối phương đã ngắt kết nối'); })		 ✅ Đã fix (Lần 2+)
ERR-04 TC-SC-14 Server trả 502 khi gọi /api/translate liên tục do Google rate limit. Không có rate limiting phía server cho endpoint translate. Fix: Cài express-rate-limit, thêm middleware:
app.use('/api/translate', rateLimit({ windowMs: 60000, max: 30 }))		 🔧 Chưa fix
ERR-05 TC-SC-22 Upload file lớn (>50MB) bị lỗi CORS trước khi đến multer middleware. CORS config thiếu header Content-Type: multipart/form-data cho preflight request lớn. Fix: Cập nhật CORS config:
app.use(cors({ origin: '*', allowedHeaders: ['Content-Type', 'Authorization'] }))
Và tăng timeout cho các upload lớn.		 🔧 Chưa fix
